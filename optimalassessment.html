Optimal Assessment

Roles

Co-founder • Designer • User researcher • Content writer

Tools

Figma

Outcome

I designed a minimum viable product (MVP) for an educational technology application that won first place in a business pitch competition. I am currently developing it into a functional MVP for improved user testing.

View final design

The situation

Even though students have varying learning styles, courses are often only taught and structured in one way. This can lead to students not being as invested in their learning as they could be. This is not the professors fault, as there are a lot of considerations that go into designing a course to meet various requirements.

Research

User research

  • I spoke with professors to learn about how they go about designing a course. One thing I learned is that different professors have different methods for designing their courses because of the specific requirements that they need to fulfill.
  • I conducted a survey to better understand the problems that professors face when designing courses. One common theme that stood out is that professors have trouble fitting all the content into a single semester.

Secondary research

  • I looked at existing research pertaining to pedagogy and how students learn. This introduced me to concepts such as Universal Design for Learning (UDL).

The design process

Wireframes

I did some early wireframes to draft out some concepts I had.

Design, test, and iterate

Once I had finished the initial design, I didn't have much time for user testing since I had to finish before the business competition (although I am planning to do more user testing in the coming months). I did want to do at least one user testing session before the competition so I could iterate based on user feedback.

User testing insight

The professor I conducted the user testing session with said that Optimal Assessment's feature that allows professors to view multiple aspects of a course (such as different learning styles, course topics, and course goals) all in one place would be really useful. However, it was presented as a virtual whiteboarding canvas, and she said she wasn't a big user of virtual whiteboards. While this was just one comment from a single user, it got me thinking about the fact that not all professors would find a whiteboarding canvas intuitive.

Iterating based on insight

This insight led me to redesign that feature with more familiar components such as checkboxes and text inputs. In the future, I might add the whiteboarding tool as an additional feature. This redesign also allowed me to combine what would have been five additional screens into a single screen, simplifying the flow.

Design decisions

UX design/information architecture

  • I chose to separate the input course information screens instead of combining it all into one screen because the different types of course information might not be related to each other in every circumstance. For example, a certain course goal might not relate to a certain course topic, but if they were both inputted on the same screen then the user might think that they need to be related.
  • On the other hand, for the "Create Assessment" screen, I chose to place all the course information checkboxes and text input boxes on the same screen because all of that information would be related to each other since it is describing a single assessment.

Visual design

  • On the course home screen, I added a line extending down the page with a dot next to each section heading to signal to the user that there is more content on the page. I added this because during the user testing session, the professor didn't realize she could scroll down the page.
  • I used green to signal primary actions to the user, such as a button that takes the user to the next page. I used black components to signify that an element will change something on the page when interacted with.

Outcome

I pitched this MVP with my partner in a business pitch competition and won first place. I am currently developing it into a functional MVP for improved user testing.
