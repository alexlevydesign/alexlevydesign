Optimal Assessment

Roles

Co-founder • Designer • User researcher • Content writer

Tools

Figma

Outcome

I designed a minimum viable product (MVP) for an educational technology application that won first place in a business pitch competition. I am currently developing it into a functional MVP for improved user testing.

View final design

The situation

Even though students have varying learning styles, courses are often only taught and structured in one way. This can lead to students not being as invested in their learning as they could be. This is not the professors fault, as there are a lot of considerations that go into designing a course to meet various requirements.

Research

Survey of professors

Secondary research

  • I looked at existing research pertaining to pedagogy and how students learn. This introduced me to concepts such as Universal Design for Learning (UDL).

The design process

Concepting

I noticed that a common theme among professors was their approaches and challenges in regards to organizing and planning course content. I created a low fidelity mockup to visualize solutions to make this process easier.

Design, test, and iterate

Due to time constraints, I only had time to do one user testing session (although a plan to do more soon).

User testing insight

The professor I conducted the user testing session with said that she liked being able to view multiple aspects of a course all in one place. However, it was presented as a virtual whiteboarding canvas, and she said she wasn't a big fan of that. While this was just one comment from a single user, it got me thinking about the fact that not all professors would find a whiteboarding canvas intuitive.

Iterating based on insight

  • This insight led me to redesign that feature with more familiar components such as checkboxes and text inputs. In the future, I might add the whiteboarding tool as an additional feature.
  • This redesign also allowed me to combine what would have been five additional screens into a single screen, simplifying the flow.

Design decisions

Component UX

  • I wanted to let the user input multiple items in a list that they can see on the same page. I designed a component that would let the user do this, but my first iteration was very confusing during user testing, as the professor had a hard time understanding how to interact with it. I simplified the component in my second iteration.

Visual design

  • On the course home screen, I added a line extending down the page with a dot next to each section heading to signal to the user that there is more content on the page. I added this because during the user testing session, the professor didn't realize she could scroll down the page.
  • I used green to signal primary actions to the user, such as a button that takes the user to the next page. I used black components to signify that an element will change something on the page when interacted with.

Outcome

I pitched this MVP with my partner in a business pitch competition and won first place. I am currently developing it into a functional MVP for improved user testing.
